# libindexer — Project Documentation

**Project:** `libindexer` — INPX library indexer + book query tool  
**Stack:** C++20, CMake, SQLite, pugixml, libzip, zlib, nlohmann/json, CLI11, Catch2  
**Dependency Management:** vcpkg (Manifest Mode)  
**Platform:** Windows (development) → Linux (production)

---

## 1. Project Overview

`libindexer` is a high-performance C++20 toolset designed to parse, index, and query large home library collections distributed in INPX format (such as librusec or flibusta archives).

The project consists of three main targets:
1. **`libindexer_core`** (Static Library): The core engine containing all the parsing, indexing, zip extraction, database, and querying logic.
2. **`libindexer.exe`** (CLI App): The executable responsible for scanning `.inpx` files, unpacking them, parsing FB2 metadata, and inserting records into the database.
3. **`libquery.exe`** (CLI App): The executable responsible for querying the generated SQLite database and returning matching books (supports JSON output).

## 3. Build Instructions

The project uses CMake and **vcpkg in Manifest Mode**.
Ensure you have `vcpkg` available and integrated with CMake.

### Setup and Build (Windows)
```powershell
# 1. Clear any old caches if you changed dependencies
Remove-Item -Force "out\build\x64-debug\CMakeCache.txt" -ErrorAction SilentlyContinue

# 2. Configure the project (vcpkg will auto-download and build dependencies)
cmake --preset x64-debug

# 3. Build the project
cmake --build --preset x64-debug
```

### Running Tests
The build will automatically invoke `tests/CreateTestData.py` (requires Python 3) to generate test ZIP archives.
```powershell
out\build\x64-debug\tests\libindexer_tests.exe
```

### Dependencies
- **vcpkg**: `nlohmann-json`, `pugixml`, `libzip`, `zlib`, `cli11`, `catch2`
- **FetchContent**: `sqlite3` (built from amalgamation source directly in CMake for specific compile definitions).

## 4. INPX Format Reference

This section documents the format parsed by `CInpParser`.

### Archive Structure
An `.inpx` file is a standard ZIP archive containing:
- One or more `.inp` files — each corresponds to one book ZIP archive (e.g., `fb2-000024-030559.inp` maps to `fb2-000024-030559.zip`).
- `collection.info` — Library metadata.
- `version.info` — Date stamp (`YYYYMMDD`).

### INP Line Format
Each line within an `.inp` file is one book record.
Fields are separated by `\x04` (byte 0x04). Exactly 15 fields per line.

| Index | Name | Notes |
|-------|------|-------|
| 0 | authors | `LastName,FirstName,MiddleName:LastName2,...:` (colon-separated, **trailing colon present**) |
| 1 | genres | `genre_code:genre_code2:` (colon-separated, **trailing colon present**) |
| 2 | title | Book title |
| 3 | series | Series name (may be empty) |
| 4 | series # | Integer string (may be empty) |
| 5 | file name | Numeric ID (e.g., `"749080"`) — file name **without extension** inside the archive |
| 6 | file size | Size in bytes |
| 7 | lib id | Internal Library ID |
| 8 | deleted | `"0"` = active, `"1"` = deleted (must be skipped) |
| 9 | file ext | Usually `"fb2"` |
| 10 | date added | `YYYY-MM-DD` |
| 11 | language | ISO code (`"ru"`, `"en"`) |
| 12 | rating | `"1"`–`"5"` or empty |
| 13 | keywords | Comma-separated or empty |
| 14 | (empty) | Always empty, artifact of the trailing delimiter — **ignore** |

## 5. Architectural Components

### Database (`CDatabase`)
A wrapper around SQLite3. Utilizes `INSERT OR IGNORE` and explicit transactions for ultimate insertion performance. Includes the `BookExists` cache and `LastInsertRowId` logic (which guards against `sqlite3_changes() == 0` on duplicates).

### ZipReader (`CZipReader`)
A static utility class that wraps **libzip**. Provides methods to list files (`ListEntries`), extract directly to memory (`ReadEntry`), and iterate over archive contents via lambdas (`IterateEntries`, `IterateEntryNames`).

### Multithreading Orchestrator (`CIndexer` in apps/indexer)
To process gigabytes of archives quickly, `libindexer` uses a concurrent Producer-Consumer queue (`ThreadSafeQueue`):
- **1 Producer Thread**: Reads the `.inpx`, parses `.inp` files, and enqueues book metadata.
- **N Consumer Threads**: Pop items off the queue, open the corresponding `.zip` file, extract the `.fb2`, parse the XML metadata using `CFb2Parser` (extracting annotations, cover images, ISBNs), and inject the fully hydrated record into `CDatabase`.

### Logging (`CLogger`)
Thread-safe singleton logger supporting `std::format`. Outputs to a specified file. (Output is strictly suppressed from `std::cout` when writing to a file, to prevent console spam during multithreaded indexing or testing).
